# Study Explain Skill

학습 이슈를 초보자도 이해할 수 있게 설명하는 댓글을 작성한다.

## 사용법

```text
/study-explain <issue_number>
```

이슈를 읽고 설명 댓글 하나를 추가한다.

## 설명 틀

### 1. 용어 사전

전문 용어를 쉬운 뜻과 일상 비유로 설명한다.

```markdown
## 용어 사전

| 용어 | 쉬운 뜻 | 일상 비유 |
|---|---|---|
| Context window | 모델이 한 번에 볼 수 있는 텍스트 범위 | 펼쳐놓을 수 있는 책상 크기 |
```

### 2. 쉬운 설명

핵심 개념을 짧게 설명한다.

```markdown
## 쉬운 설명

### [개념]이 무엇인가.

**한 줄 요약**: [10단어 이내]

**비유**:
[3-4문장]

**왜 도움이 되는가**:
[해결하는 문제]
```

### 3. 그림

ASCII diagram을 사용한다.

```markdown
## 그림으로 이해하기

### 이전
[문제 상태]

### 이후
[개선 상태]

### 흐름
[단계별 화살표]
```

### 4. 단계별 설명

과정을 한 단계씩 분해한다.

```markdown
## 단계별로 따라가기

**상황**: [구체적인 예시]

**1단계: [행동]**
- 일어나는 일:
- 이유:
- 예시:
```

### 5. 왜 중요한가

이 자료가 무엇을 가능하게 하는지 설명한다.

### 6. 자주 하는 오해

잘못된 이해와 올바른 이해를 분리한다.

### 7. 연결해서 이해하기

이미 아는 개념이나 관련 이슈와 연결한다.

### 8. 한 장 요약

마지막에 ASCII 박스로 요약한다.

## 출력 형식

```markdown
# [학습 제목] 쉬운 설명

> 이 댓글은 이슈 본문을 더 쉽게 읽을 수 있도록 다시 설명한다.

[위 섹션]

---
이 설명은 `/study-explain` 흐름으로 작성했다.
```

## 품질 체크리스트

- [ ] 중요한 용어가 설명되어 있다.
- [ ] 비유가 장식이 아니라 이해를 돕는다.
- [ ] ASCII diagram이 읽힌다.
- [ ] 단계별 설명에 이유가 포함되어 있다.
- [ ] 오해 방지가 명시되어 있다.
- [ ] 요약이 짧다.
