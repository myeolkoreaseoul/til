# til

배우고 만들면서 알아낸 것을 GitHub Issue로 정리한다.

주제는 회계, 정산, 세무, 규정 자동화, AI 에이전트, 데이터다.

## 운영 방식

이 저장소는 공개 가능한 기술 자료만 다룬다.

학습 흐름은 다음과 같다.

```text
Discover -> Check -> Scan -> Analyze -> Document -> Explain -> Connect -> Close
```

| 단계 | 산출물 |
|---|---|
| Check | 게시 적합성 점검 |
| Scan | 읽을 가치와 우선순위 |
| Analyze | 깊은 분석 |
| Document | GitHub Issue 본문 |
| Explain | 쉬운 설명 댓글 |
| Connect | 관련 이슈 연결 |

세부 규칙은 `CLAUDE.md`와 `.claude/skills/`를 따른다.

## 표현 원칙

문서는 한국어로 쓴다.

공개 여부와 무관하게 특정 대상의 방식을 그대로 가져온다는 인상을 주는 표현을 쓰지 않는다.

기술 자료를 다룰 때는 `참고`, `분석`, `도입 후보`, `적용 가능 요소`, `보류할 요소`처럼 중립적인 표현을 쓴다.
